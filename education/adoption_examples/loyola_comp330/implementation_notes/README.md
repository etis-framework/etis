# Implementation Notes

## Purpose

This directory preserves institutional memory for the Loyola University Chicago COMP330/474 implementation.

The purpose is to capture observations that emerge from repeatedly teaching and evolving the course over time.

This directory intentionally remains lightweight.

These notes are not educational doctrine.

These notes are implementation memory.

Over time, these observations may help improve both the course and ETIS itself.

---

## Core Philosophy

Educational systems improve when observations are intentionally preserved.

Small observations often become important insights.

Educational memory should not be recreated every semester.

Educational memory should accumulate.

This directory exists to preserve those observations.

---

## Relationship To ETIS

This directory contributes to the bi-directional feedback system that connects ETIS and COMP330/474.

```text
ETIS

↓

COMP330/474

↓

Educational Observations

↓

Educational Memory

↓

ETIS Refinement

↓

Future Implementations
```

These notes help ensure that educational experiences continuously improve ETIS.

---

## What This Directory Captures

| File | Purpose |
|------|---------|
| `semester_observations.md` | High-level observations from repeated semester implementations |
| `student_patterns.md` | Common student behaviors and learning patterns |
| `repository_patterns.md` | Common repository behaviors and engineering maturity patterns |
| `adoption_insights.md` | Insights useful to future ETIS adopters |

These files should remain concise.

The goal is long-term maintainability.

---

## Writing Philosophy

Observations should be:

* short
* practical
* evidence based
* actionable
* timeless when possible

Avoid semester-specific details unless they reveal important trends.

Capture patterns rather than isolated events.

Focus on observations that may benefit future instructors.

---

## What Does Not Belong Here

Do not duplicate:

* syllabi
* schedules
* assignments
* rubrics
* ETIS doctrine
* educational philosophy documents

Those artifacts already exist elsewhere.

This directory should preserve implementation memory only.

---

## Review Questions

Instructors should periodically ask:

* What patterns are emerging?
* What consistently works well?
* What repeatedly creates friction?
* What surprised us?
* What should future instructors know?

---

## Engineering Standard

Educational memory should be lightweight enough that instructors will actually maintain it.

Strong educational systems preserve observations rather than recreating them each semester.
