# ETIS Instructor Course Package

The **ETIS Instructor Course Package** provides instructor-facing guidance for teaching *Engineering Trustworthy Intelligent Systems (ETIS)* in academic, professional, and organizational learning environments.

This package exists to help instructors adopt ETIS as a teachable educational system rather than a book, topic list, or collection of documents.

ETIS education prepares learners to:

- define intent
- engineer context
- bound authority
- verify behavior
- operate reality
- explain decisions
- own outcomes

Educational work should resemble professional engineering work.

The Instructor Course Package operationalizes that philosophy.

---

# Purpose

The purpose of the Instructor Course Package is to help instructors assemble, teach, operate, adapt, and steward ETIS educational systems.

The package helps instructors answer practical questions such as:

- How should ETIS be introduced?
- How should learning experiences be sequenced?
- How should assignments increase engineering maturity?
- How should AI-assisted work be governed?
- How should students be assessed?
- How should classroom experiences be facilitated?
- How should educational systems be stewarded over time?

This package does not replace the ETIS book.

The ETIS book remains authoritative.

This package translates ETIS doctrine into teachable educational systems.

# Course Design Philosophy

ETIS courses should be designed around engineering maturity rather than topic coverage.

Students should practice:

- requirements discipline
- planning
- architecture reasoning
- decision documentation
- AI governance
- implementation under review
- testing and validation
- evidence creation
- release readiness
- operational thinking
- governance
- engineering defense

Students should become accountable for both what they build and the evidence they create.

---

# AI Governance Philosophy

ETIS does not ask students to avoid AI.

ETIS asks students to govern AI.

The governing principle remains:

> AI-assisted work is not accepted engineering work until humans review, verify, and own it.

AI should be normalized throughout educational systems.

AI avoidance should not be assessed.

AI responsibility should be assessed.

Before teaching ETIS, instructors should be able to answer:

- What transformation path am I teaching?
- Which ETIS chapters are primary?
- What student engineering environment will be used?
- What educational engines will be used?
- What shared assets will be consumed?
- How will engineering maturity increase?
- How will AI be governed?
- How will students practice engineering?
- How will students defend engineering decisions?
- How will educational systems be improved over time?

The course is ready when these answers are clear.

# Constitutional Educational Pillars

These principles should remain stable.

- Educational work should resemble professional engineering work.
- Engineering accountability is the educational outcome, not the side effect.
- Educational systems are engineered. Educational systems are also stewarded.
- Educational systems are inherited, not reinvented.
- Educational memory is educational infrastructure.
- Every semester should leave evidence for the next instructor.
- AI usage is not an academic violation. Undisclosed and unverified AI dependency is an engineering risk.
- Do not assess AI avoidance. Assess AI responsibility.
