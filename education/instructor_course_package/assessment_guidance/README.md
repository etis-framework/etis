# ETIS Assessment Guidance

The **ETIS Assessment Guidance** directory provides reusable frameworks for evaluating engineering maturity throughout ETIS educational experiences.

This directory intentionally moves beyond traditional grading systems.

ETIS assessment focuses on professional accountability rather than assignment completion.

The objective is not to determine whether students can build software.

The objective is to determine whether students can engineer systems that can be understood, reviewed, governed, operated, improved, and trusted over time.

Assessment should progressively resemble professional engineering evaluation.

---

## Purpose

The purpose of this directory is to help instructors build assessment systems that reinforce trustworthy engineering.

This directory helps instructors answer questions such as:

- What should be assessed?
- How should assessment evolve over time?
- How should engineering accountability be evaluated?
- How should evidence be evaluated?
- How should AI governance be evaluated?
- How should release readiness be evaluated?
- How should operational thinking be evaluated?
- How should students defend engineering decisions?

The objective is to evaluate engineering maturity rather than technical output alone.

---

## Educational Philosophy

Traditional software engineering assessment often emphasizes outputs.

Examples include:

```text
Homework

↓

Projects

↓

Presentations

↓

Exams
```

ETIS intentionally expands assessment.

Students should also be evaluated on:

```text
Intent

↓

Evidence

↓

Architecture

↓

Review Participation

↓

AI Accountability

↓

Release Readiness

↓

Operational Thinking

↓

Professional Ownership
```

Educational assessment should progressively resemble professional engineering evaluation.

---

## Guiding Principle

ETIS assessment should answer one question:

> What engineering responsibility should learners now be able to own?

Assessment should progressively increase ownership throughout the educational experience.

Students should evolve from completing work to defending engineering outcomes.

## What Does Not Belong Here

This directory should remain reusable and implementation agnostic.

The following artifacts should not be placed here:

- institutional gradebooks,
- LMS exports,
- semester-specific grading spreadsheets,
- instructor-specific point systems,
- local administrative policies,
- registrar policies,
- semester logistics.

Those artifacts belong inside local implementations.

For example:

```text
education/
└── adoption_examples/
    └── loyola_comp330/
```

## Adaptation Philosophy

Educational environments have different constraints.

Examples include:

- enrollment size,
- calendar structures,
- learner maturity,
- instructional modalities,
- institutional requirements.

These constraints may alter implementation.

They should not alter ETIS doctrine.

---

## Stewardship Rules

Future maintainers should preserve these rules.

- Do not reduce assessment to grading.
- Do not assess technical output alone.
- Do not disconnect assessment from evidence.
- Do not disconnect assessment from reviews.
- Do not ignore AI accountability.
- Do not disconnect assessment from operations.
- Do not disconnect assessment from professional ownership.

This directory should remain reusable, maintainable, and cumulative.

---

## Guiding Standard

Every file in this directory should answer at least one of these questions:

- What engineering behavior are we evaluating?
- What evidence supports evaluation?
- What decisions should students explain?
- What risks should students communicate?
- What accountability should students demonstrate?
- What ownership should students assume?

If a file cannot answer one of those questions, it probably does not belong here.

---

## Core Commitment

The purpose of ETIS Assessment Guidance is not to create stronger grading systems.

The purpose is to create stronger engineers.

Assessment should progressively help students learn how to define intent, engineer context, bound authority, verify behavior, operate reality, explain decisions, and own outcomes.

That progression is one of the foundations of ETIS education.
