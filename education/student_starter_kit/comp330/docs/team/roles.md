# Roles

## Purpose

This document defines team responsibilities and ownership areas.

Roles establish ownership.

Roles do not create silos.

Everyone contributes to the project.

Ownership simply ensures accountability remains visible.

---

## Team Roles

| Role | Primary Responsibilities | Owner | Backup |
|------|-------------------------|-------|--------|
| Team Lead | Coordination and communication | | |
| Requirements Lead | Requirements and acceptance criteria | | |
| Architecture Lead | Architecture and technical decisions | | |
| Development Lead | Implementation coordination | | |
| Quality Lead | Testing and validation | | |
| Release Lead | Release readiness and evidence | | |

---

## Ownership Expectations

Role owners should:

- coordinate work
- surface risks
- communicate blockers
- review evidence
- support teammates

---

## Shared Responsibilities

Everyone is responsible for:

- code quality
- reviews
- AI governance
- engineering evidence
- professional conduct

---

## Review Questions

- Is ownership balanced?
- Are bottlenecks emerging?
- Does everyone have visible responsibilities?

---

## Engineering Standard

Ownership should be visible.

Ownership should never be isolated.
