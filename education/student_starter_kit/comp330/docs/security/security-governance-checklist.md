# Security Governance Checklist

## Purpose

This checklist helps teams identify obvious security risks before release.

This is not an enterprise security audit.

It is a lightweight engineering review.

---

## Checklist

### Data

- [ ] Data collection is necessary.
- [ ] Sensitive data are identified.
- [ ] Synthetic data are used when possible.

### Permissions

- [ ] Access boundaries are documented.
- [ ] Permissions are intentionally assigned.
- [ ] Excessive privileges are avoided.

### AI

- [ ] AI outputs have been reviewed.
- [ ] AI risks have been considered.

### Release

- [ ] Known limitations are documented.
- [ ] Remaining risks are visible.

---

## Review Questions

- What obvious risks remain?
- What assumptions need further review?
- What should be deferred?

---

## Engineering Standard

Strong engineering teams make risks visible before release.
