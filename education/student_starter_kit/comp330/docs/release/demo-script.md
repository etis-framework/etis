The goal is not performance.

## Required Talking Points

During the demo, explain:

- what problem the system addresses
- what is included in this release
- what evidence supports the release claims
- how AI was used and verified
- what limitations remain
- what risks remain
- what would improve next

## Review Questions

- Does the demo match the release notes?
- Are claims supported by evidence?
- Are limitations disclosed?
- Can the team defend what is shown?

## Engineering Standard

A demo should communicate evidence, not hide uncertainty.
