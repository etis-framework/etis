# Release Package

## Purpose

This directory helps teams prepare evidence for release readiness.

A release is not just a demo.

A release is an engineering claim that the system is understandable, reviewed, tested, bounded, and defensible.

Keep release evidence clear and practical.

---

## Release Artifacts

| Artifact | Purpose |
|----------|---------|
| `release-notes.md` | Summarizes what changed and what is included |
| `known-limitations.md` | Documents limitations, risks, and deferred work |
| `traceability-summary.md` | Connects release claims to evidence |
| `demo-script.md` | Guides a defensible demonstration |

---

## Relationship To ETIS Philosophy

A demo is not operational proof.

Students should ask:

- What are we claiming?
- What evidence supports the claim?
- What remains limited?
- What risks remain?
- What can we defend?

---

## Engineering Standard

A release is not ready because it works once.

A release is ready when its claims can be defended with evidence.
