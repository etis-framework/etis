# Data Package

## Purpose

This directory contains project data used for development, testing, demonstration, or analysis.

Data should support engineering work.

Data should not become a dumping ground.

Keep data small, intentional, and understandable.

---

## Appropriate Uses

Examples include:

- sample data
- synthetic test data
- seed data
- demonstration data
- small reference datasets

---

## Data Rules

Do not store:

- private data
- student records
- institutional data
- secrets
- credentials
- large generated files
- production data

Use synthetic data whenever possible.

---

## Documentation Expectations

If data are added, document:

- what the data represent
- where the data came from
- whether the data are synthetic
- how the data should be used
- any limitations

---

## Relationship To ETIS Philosophy

Data creates engineering responsibility.

Teams should ask:

- What data are we using?
- Why do we need it?
- Is it safe to store?
- Is it synthetic or real?
- Could another engineer understand it?

---

## Engineering Standard

Use only the data needed to support responsible engineering work.
